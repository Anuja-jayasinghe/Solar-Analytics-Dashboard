## Directory rules
- Each logic gets its own self-contained file.
- Add new entries only after they are confirmed and implemented.
- Keep this README short.

## File structure guide
Each logic file should be readable on its own and should follow this order:
1. Title and status
2. Owner and primary files
3. Purpose or business intent
4. Data sources
5. Canonical rules
6. How it works technically
7. Worked examples
8. Formula summary or calculation notes
9. Implementation checklist
10. Edge cases
11. Acceptance criteria
12. Current implementation files

Keep the language simple, explicit, and implementation-ready so a human, an automated system, or an AI agent can rebuild the logic from the file without extra context.
